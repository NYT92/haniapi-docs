# Search

This API can use it to search for videos, description, tags, and more. You can sort the results with order_by request too.

| Body Params | Description                                                                                             |
| ----------- | ------------------------------------------------------------------------------------------------------- |
| search      | Search Title, Description, Tags, etc                                                                    |
| tags        | Find Video with tags [Tags](#tags)                                                                      |
| brands      | Find all of brands [Brands](#brands)                                                                    |
| blacklist   | Exclude tags on search [Blacklist](#blacklists)                                                         |
| order_by    | Sort all of the search [Order_by](#order)                                                               |
| ordering    | Ascending and Descending search [Order_by](#order)                                                      |
| page        | Get all pages search request (if the page have multiple page all of the array will start from 0 to 50+) |

## Using Body Params

The reason that we use body params is because we want to make the search more flexible and easy to use. ALSO
you wont waste your time to input the search query in the url.

<Callout>
  Those request can be blank and it still giving you search results.
</Callout>

For Non Self host User : Before using the search, Read this [Info](docs/info#usage) before you can use it.

Body Params :

```req
{
    "search": "",
    "tags": [],
    "brands": [],
    "blacklist": [],
    "order_by": "",
    "ordering": "",
    "page": 0
}
```

## Using Query Params

If you are not using the body request, you can use the query parameters to get the search.  
This routes is limited so you cannot add tag, blacklist and brands search

<Callout>
  Those request can be blank and it still giving you search results.
</Callout>

For Non Self host User : Before using the search, Read this [Info](info#usage) before you can use it.

> /search/req

Available Query Parameters :

````sr
q= Search Query  
p= Page  
ordering=  Ordering from the search (asc or desc)  
order_by=  Order_by type of item
````

## Error

Wrong Typo Might be caused by the search body request.  
It's will show the 500 error message.

Check the body request before requesting to the server.
